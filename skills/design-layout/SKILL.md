# design-layout — 정규화 레이아웃·UX

> 출처 합성: Dammyjay93/interface-design(5.2k★)의 토큰 고정·드리프트 방지 구조 + nextlevelbuilder/ui-ux-pro-max(100k★)의 수치 규칙 발췌. 화려한 비주얼 지향인 Anthropic frontend-design은 목적이 달라 채택하지 않음.
> 전부 수치 규칙이라 어떤 모델이든 그대로 적용하면 된다.

## 핵심 구조: 토큰은 파일에 고정한다

프로젝트에서 처음 UI를 만들 때 `docs/design-tokens.md`를 생성하고, **이후 모든 UI 작업 전에 이 파일을 먼저 읽는다.** 세션이 바뀔 때 버튼 높이가 36→38→40px로 흘러가는 드리프트가 이 파일 하나로 막힌다. 파일에 없는 결정을 새로 내리면 파일에 추가한다.

토큰 파일 최초 생성 시 방향을 한 번만 묻는다: "① 정밀·밀도형(대시보드, 도구) ② 따뜻·친근형(콘텐츠, 아이들 대상)" — 이후 팔레트·radius·간격 밀도가 여기서 갈린다.

## 기본 토큰 (프로젝트 토큰 파일이 없을 때의 기준값)

### 타이포그래피
- 본문 16px 고정 (iOS 자동 줌 방지). 타입 스케일: 12 / 14 / 16 / 18 / 24 / 32px — 이 6단계 밖의 크기 금지.
- 행간 1.5~1.75. 한 줄 길이 데스크톱 60~75자.
- 한국어: `word-break: keep-all` (단어 중간 줄바꿈 방지), 본문 `letter-spacing: -0.01em` 내외.

### 간격·레이아웃
- 모든 간격은 4/8px 배수: 4, 8, 12, 16, 24, 32, 48. 임의 값(예: 13px, 22px) 금지.
- 섹션 수직 간격 계층: 관련 요소 16 → 그룹 24 → 섹션 32~48.
- 버튼 높이 36px(터치 대상이면 44px), 패딩 12×16px. 카드 패딩 16px, radius 8px.
- 깊이 표현은 하나만 골라 통일: "보더만" 또는 "표면 명도 단계" — 그림자·보더·명도 혼용 금지.

### 반응형
- 375px 모바일 우선 → 768 → 1024 → 1440px. 모바일에서 body 가로 스크롤 금지 (넓은 표는 컨테이너 내부 스크롤).
- 터치 타깃 44×44px 이상, 타깃 사이 8px 이상.

### 색·접근성
- 팔레트는 시작할 때 4~6개 hex로 명명해 고정 (background / surface / text / muted / primary / accent). 작업 중 새 색 즉흥 추가 금지.
- 대비: 본문 4.5:1 이상, 큰 글자 3:1 이상 (WCAG AA).

### 모션
- 전환 150~300ms. 400ms 초과 금지. hover에 레이아웃이 밀리는 효과 금지.

## 안티패턴 (발견 즉시 수정)

- placeholder만 있고 label 없는 폼 필드.
- 포커스 링 제거 (`outline: none` 단독).
- 이모지를 아이콘 대용으로 사용.
- 과한 애니메이션·그라데이션 — 오히려 AI가 만든 티가 난다. 대담함은 한 곳에만, 주변은 조용하게.

## 작업 절차

1. `docs/design-tokens.md` 읽기 (없으면 방향 질문 → 위 기준값으로 생성).
2. UI 작성 — 토큰 밖 값이 필요해지면 임의로 쓰지 말고 토큰에 추가할지 판단.
3. 완료 후 자가 점검: 간격이 전부 4/8 배수인가, 폰트가 6단계 안인가, 대비 4.5:1인가, 375px에서 가로 스크롤 없는가.
