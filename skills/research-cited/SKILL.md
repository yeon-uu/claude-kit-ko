# research-cited — 출처 있는 자료조사

> 출처 합성: jamditis/claude-skills-journalism(주장 추출·증거 강도·반대 증거 의무) + daymade/deep-research(출처 등급·[unverified]·AS_OF) + 199-biotechnologies(사실/추정 구분 기록)를 경량화하고 한국 출처 계층을 추가.
> 규칙 기반이라 어떤 모델이든 적용 가능. 검색 없이 기억으로 수치를 쓰는 것이 가장 흔한 위반이다.

## 철칙

산출물에 들어가는 모든 사실 문장은 셋 중 하나여야 한다:
1. **출처 링크가 붙은 사실** — [n] 인라인 인용
2. **`[추정]` 표기 + 계산 근거** — "추정: 공표된 X와 Y로 계산"
3. **"확인 못 함"** — 아는 척하지 않고 그대로 쓴다 (`[unverified]`)

기억 속 수치를 출처 없이 쓰지 않는다. 학습 데이터의 수치는 낡았거나 틀렸을 수 있다.

## 출처 등급 (한국 기준)

| 등급 | 출처 | 용도 |
|---|---|---|
| 1 | 정부·공공 원데이터: 공공데이터포털, KOSIS(통계청), 국가법령정보센터, 각 부처 보도자료·공고 원문, KDI·한국은행 | 핵심 수치는 여기서. 보고서의 주요 주장 ≥30%는 1등급 출처 |
| 2 | 학술: 논문(DOI), 학회, RISS·DBpia / 공식 기술문서 | 방법론·기술 사실 |
| 3 | 언론 보도, 업계 리포트 | 사건·동향. 수치는 원출처(1·2등급) 역추적 시도 |
| 4 | 블로그, 커뮤니티, SNS, 위키 | 단서로만. 단독 인용 금지 — 반드시 상위 등급으로 확인 |

## 절차

### 1. 주장 추출
조사 질문을 "검증 가능한 사실 주장" 단위로 쪼갠다. 의견·예측은 검증 대상에서 제외하고 의견임을 표기. 주장 유형별 1차 출처: 통계→정부 원데이터, 법·규정→법령정보센터, 인용문→원문.

### 2. 검색·수집
- 주요 주장은 **독립 출처 2개 이상** (같은 보도자료를 받아쓴 기사 2개는 1개로 센다).
- 시간 민감 주장은 기준일 표기: "(2026-07 기준)". 낡은 수치는 최신 확인 후 사용.
- 접근 못 한 출처(유료 논문 등)는 초록·2차 인용으로 썼음을 표기.

### 3. 반대 증거 의무 탐색
결론을 굳히기 전에 반드시 1회: 결론과 모순되는 출처를 검색한다 ("X 효과 없음", "X 논란", "X 반박"). 나오면 본문에 병기하고, 안 나오면 "반대 증거 검색했으나 미발견"을 남긴다. 확증만 모으는 조사가 가장 흔한 실패다.

### 4. 산출물 형식
- 인라인 [n] 인용 + 문서 끝 출처 목록 (제목, 기관, 날짜, URL).
- 사실/추정/의견이 시각적으로 구분되게: 추정에는 `[추정]`, 미확인에는 `[unverified]`.
- 신뢰도 요약 한 줄: "핵심 수치 N개 중 1등급 출처 M개, 추정 K개."

## 자가 점검 (제출 전)

- 출처 없는 수치가 한 개라도 있는가 → 있으면 미완성.
- 4등급 출처가 단독 근거인 주장이 있는가.
- 모든 주장이 같은 방향인가 → 반대 증거 탐색을 건너뛴 신호.
