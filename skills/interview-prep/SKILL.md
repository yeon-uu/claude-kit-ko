# interview-prep — 면접 대비·질문은행

> 근거: 다대다 면접 실측 기준(링커리어 합격자료 — 1인당 답변 30초~1분, 공통 질문 4~6문항, 5대 질문 유형). 경험질문 재료는 experience-block이 쌓은 블록을 사용한다.
> 권장 순서: jd-fit 리포트가 있으면 그걸 입력으로 (기업 리서치·갭이 이미 분석됨). 없으면 회사·직무명만으로 진행하되 정밀도가 낮음을 고지.

## 0. 형식 확인 (첫 질문)

- **다대다** → 30초 버전 중심 (공통 질문에 4~6명이 답하므로 길면 잘림)
- **1:1 심층** → 60초 버전 + 꼬리질문 2단계
- **모름** → 둘 다 준비, 30초 우선

## 1. 예상질문 3종 세트

### A. 일반질문 (인성·공통 5대)
1. **자기소개** — 30초·1분 두 버전, 직무 키워드 중심 (블록 INDEX의 키워드에서).
2. **지원동기** — jd-analysis 리포트의 기업 리서치 + 내 스토리 1개 연결. 일반론 금지.
3. **팀 갈등·협업 경험** — 블록에서 협업 스토리 선별(STAR). 없으면 지어내지 말고 "협업 스토리 갭"으로 고지 → 동아리·팀플에서 발굴 인터뷰.
4. **최근 직무 관련 이슈** — research-cited 기준으로 실제 이슈 1~2개 조사(출처 링크), 직무와의 연결 한 줄.
5. **입사 후 포부** — 공고의 팀 업무 기반으로 구체적으로. "열심히 하겠습니다"류 포부 상투어 금지 (write-human).

### B. 짧은질문 (팩트형 — 기술 단답)
- JD 요구사항과 내 키워드의 교집합에서 **CS·기술 단답 10~15개** 도출 (예: "인덱스가 왜 빠른가", "async가 필요한 순간은", "JWT 탈취되면?", "트랜잭션 격리 수준").
- 각 질문에 **3문장 이내 모범답** + 가능하면 내 프로젝트 연결 한 줄 ("실제로 AI-DO에서 ~했습니다" — 단답을 경험 증거로 승격시키는 장치).
- **모르는 질문 대응 원칙을 함께 리허설**: 아는 데까지 답하고 "여기부터는 확인해 보겠습니다" — 아는 척이 최악의 수다.

### C. 경험질문
- 관련 블록의 STAR 30초/60초 그대로 + **예상 꼬리질문 2단계** ("왜 그 방법을?", "다시 한다면?") + **약점 질문** ("왜 Redis 안 썼어요?" — 블록의 기각한 대안이 답의 재료).

## 2. 산출물

`_career\interview-prep\YYYY-MM-DD-<회사 또는 직무>.md` — 3종 세트 + 리허설 체크리스트(30초 버전을 실제로 소리 내어 시간 재기). 다대다면 "다른 지원자 경청 태도, 순서 외 발언 지양" 팁 첨부.

## 3. 회고 모드 (면접 직후 — "면접 봤어" 트리거)

1. **받은 질문 수집** — 대화로 기억나는 질문을 전부 받아 적고 `_career\interview-bank.md`에 기록: 질문 / 유형(일반·팩트·경험) / 예상 적중 여부 / 내 답 요약 / 아쉬운 점.
2. **답 못 한 기술 질문 → gap-plan.md에 추가** (갭의 최고 등급 증거는 "면접에서 실제로 못 답한 것").
3. `applications.md` 해당 행에 결과·회고 링크 갱신.

질문은행이 쌓일수록 다음 대비의 1순위 예상질문은 추측이 아니라 **실제 기출**이 된다 — 이 루프가 이 스킬의 핵심이다.

## 안티패턴

- 블록에 없는 경험으로 답변 대본 작성 (사실 보존 — 갭은 갭이라고 말한다).
- 모범답을 암기 문어체로 작성 — 말로 뱉을 수 있는 구어체로 (write-human).
- 예상질문만 만들고 회고를 안 하는 것 — 은행 없는 대비는 매번 제자리다.
