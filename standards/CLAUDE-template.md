# 프로젝트 CLAUDE.md 템플릿

프로젝트 CLAUDE.md는 **1페이지**를 넘기지 않는다. 사용자 선호·범용 가드레일은 상위 계층(전역 CLAUDE.md, standards)에 있으므로 쓰지 않는다. 여기엔 "이 프로젝트에서만 참인 것"만 쓴다.

아래를 복사해서 시작:

```markdown
# CLAUDE.md — <프로젝트명>

> 한 줄 설명 (무엇을 만드는가)

## 상태
<활성 | 완료·포폴 모드 | 보류 | 아카이브> — <날짜 기준 한 줄 근거>

## 목적 분류
<공모전·논문·포폴용 | 상업 출시용 | 연습·실험용>
<!-- 분류에 따라 기준이 다름: 공모전용=심사기준 역산·마감 준수, 상업용=운영·과금·법적 이슈, 연습용=학습 목표 달성이 완성 -->

## 스택·구조
- 스택: <예: FastAPI + PostgreSQL 16 + Docker Compose, EC2 t3.small>
- 설계 문서: <docs/ARCHITECTURE.md 등 진실 소스 지정>
- 구조: <핵심 디렉터리 3~6줄>

## 이 프로젝트의 규칙
<!-- 범용 규칙(standards/backend-guardrails.md)은 링크만. 여기엔 프로젝트 고유 규칙만 -->
- <예: SSE 응답은 nginx proxy_buffering off 필수>
- <예: 외부 API X가 주 데이터 소스, 나머지는 보조>

## 알려진 함정
<!-- 이 프로젝트에서 실제로 겪은 것만. 범용 함정은 backend-guardrails.md로 -->

## 지금 할 일 / 하지 말 것
- 할 일: <현재 단계>
- 하지 말 것: <스코프 밖 항목 명시 — 과잉 생성 방지>
```

## 상태 값 정의

| 상태 | 의미 | 에이전트 행동 |
|------|------|------|
| 활성 | 현재 개발 중 | 정상 작업 |
| 완료·포폴 모드 | 개발 끝, 포폴 자산 | 코드 변경 최소화. README·경험블록 정비 위주 |
| 보류 | 재개 예정 | 재개 조건을 CLAUDE.md에 명시 |
| 아카이브 | 다시 안 건드림 | 읽기 전용. 수정 요청 시 사용자에게 상태 확인 |
