# 에이전트 메모리 정형 포맷

Claude Code의 프로젝트 메모리(`~\.claude\projects\<프로젝트>\memory\`)에 적용한다.

## 파일 포맷 (1파일 = 1사실)

```markdown
---
name: <kebab-case-슬러그>          # 파일명과 동일, 밑줄(_) 금지
description: <한 줄 요약 — 세션 시작 시 회상 판단에 쓰임>
metadata:
  type: user | feedback | project | reference
---

<사실 본문. 관련 메모리는 [[슬러그]]로 링크>
```

- `feedback` 타입은 본문에 **Why:** (왜 이 지적이 나왔나) / **How to apply:** (다음에 어떻게 적용하나) 줄을 붙인다.
- 상대 날짜("다음 주") 금지 — 절대 날짜(2026-08-07)로 변환해 저장.
- 저장 후 `MEMORY.md`에 한 줄 인덱스 추가: `- [제목](파일.md) — 후크`. MEMORY.md에 본문을 쓰지 않는다.

## 무엇을 저장하고, 무엇을 저장하지 않는가

| 저장한다 | 저장하지 않는다 (대신 갈 곳) |
|---|---|
| 사용자가 준 작업 방식 피드백 | 사용자 정체성·선호 — **원본은 `~\.claude\CLAUDE.md`**, 프로젝트 메모리에 프로필 중복 저장 금지 |
| 코드·git에서 유도할 수 없는 프로젝트 사실 (외부 마감일, 팀 합의, 실패 교훈) | 진행상황·자잘한 결정 → 프로젝트 `docs\DEVLOG.md` |
| 외부 리소스 포인터 (대시보드 URL, 공고 링크) | 코드 구조·과거 수정 내역 → git·CLAUDE.md가 이미 기록 |

- **최소한만 저장, 저장 전 사용자에게 확인.**
- 틀린 것으로 판명된 메모리는 갱신이 아니라 삭제 대상인지 먼저 판단한다.
- 분기마다 한 번 `consolidate-memory` 스킬로 중복 병합·낡은 사실 정리.

## 프로젝트 이동·아카이브 시

프로젝트 폴더를 옮기면 메모리 디렉터리 경로가 끊긴다(고아 메모리). 이동 전에 보존할 사실만 새 경로의 메모리로 옮기고, 옛 디렉터리는 삭제한다.
